# Daily Workflow — Run This Every Session, In Order

The instruction set the operating brain executes at the start of every session. Same network-standard shape, with one structural difference: **the goals pulse runs across three funnels, not one or two.**

---

## STEP 1 — FEEDBACK CHECK (Always First)

Open `feedback/feedback-log.md` AND scan `data/inbound-emails/` for anything new. Process each entry: identify the affected file, propose the edit, get approval, log in `improvements.md`.

---

## STEP 2 — DATA INTAKE

- New training-client athlete file → scaffold `output/reports/athletes/`.
- New managed-athlete (NFL/P4) update → route through `skills/athlete-brand-pipeline/SKILL.md`.
- New session log → extract what landed, update athlete file.
- New parent/coach/trainer-candidate inbound → route to the matching comms skill.
- New advocacy call scheduled or completed → route through `skills/cbh-advocacy/SKILL.md`.
- New NIL deal activity → update `data/nil-deals/`.
- New engagement scrape → hold for Monday review, don't process daily.

---

## STEP 3 — THREE-FUNNEL GOALS PULSE

Unlike single-funnel repos, this step explicitly checks all three funnels plus the pre-August sprint goals:

> **Funnel 1 (Athlete/CBH):** Did anything move forward? Blocking?
> **Funnel 2 (Trainer/Blueprint):** Did anything move forward? Blocking?
> **Funnel 3 (Athlete-Brand/NIL):** Did anything move forward? Blocking?
> **Pre-August Sprint (P1-P7, if before August 10):** Status on the active goal(s) for this week?

Output a per-funnel + per-sprint-goal pulse at the top of the session log.

---

## STEP 4 — TODAY'S PRODUCTION

Priority order when multiple categories apply during the pre-August sprint: **Readiness Plan gap-closing work** → parent/athlete comms → advocacy brief prep → content → outreach → other projects.

After August 10, priority order shifts to: parent/athlete comms → active event/cohort work → advocacy → content → reports → outreach → projects.

- **CBH content day** → `skills/margin-voice/SKILL.md` + `skills/subject-media-coordination/SKILL.md` → brief, don't produce final assets.
- **Trainer Blueprint day** → `skills/trainer-blueprint/SKILL.md` → application processing, candidate outreach prep, curriculum work.
- **Athlete-brand-pipeline day** → `skills/athlete-brand-pipeline/SKILL.md` → weekly cadence per managed athlete.
- **Advocacy day** → `skills/cbh-advocacy/SKILL.md` → pre-call briefs, debrief capture.
- **Parent comms day** → `skills/parent-comms/SKILL.md`.
- **Report day** → weekly/monthly reports → `output/reports/`.

---

## STEP 5 — CLOSE THE LOOP

- [ ] Everything saved to `output/` with today's date.
- [ ] One-paragraph session log appended to `output/reports/session-log.md`.
- [ ] List anything waiting on Kyron or Margin — approvals, missing data, decisions.

---

## Special Routine — Monday Mornings

`skills/engagement-monitoring/SKILL.md` runs end-to-end across all three funnels. `skills/athlete-brand-pipeline/SKILL.md`'s weekly roster pulse also runs Monday — these are two distinct reports, not duplicates.

---

## Special Routine — Pre-August Sprint (Now Through August 10)

Every session, in addition to the standard steps: check `STL-10-of-10-Readiness-Plan.md` for any component that moved status this session. Update the status mark if so. This file is the single source of truth for "are we ready" — keep it current every session, not just weekly.

---

*If a session ends and nothing new exists in the repo, the session failed. Files are the proof of work.*
